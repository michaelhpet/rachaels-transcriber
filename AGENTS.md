## Critical Architecture Rules

### Backend (Rust)
1. **Keep system/file operations in Rust** — use `rfd::FileDialog` for dialogs, Rust stdlib for file I/O. Only add a `tauri-plugin-*` when Rust-side alternatives are impractical.
2. **Progress streamed via events** — long-running ops spawn `std::thread::spawn` and emit Tauri events. Frontend listens via `listen()`.
3. **AppState** in `lib.rs` — 7 fields: `cancel_flag` (AtomicBool), `engine` (Mutex\<Option\<WhisperEngine\>\>), `recorder` (Mutex\<Option\<AudioRecorder\>\>), `save_audio_path`, `save_transcript_path`, `vad_enabled` (AtomicBool), `samples_processed` (AtomicUsize, tracks raw interleaved position).
4. **Model storage** — uses `app.path().app_data_dir() -> /models/`. Set during Tauri `.setup()` hook. Falls back to `./models` for CLI mode.
5. **Model autodownload** — in `App.tsx` `useEffect` on mount: `checkModels()` → if any missing, `downloadModels()`. No user download buttons.
6. **Recording chunks** — non-overlapping: `read_chunk(consumed_raw, WINDOW_SEC)` reads raw audio at actual mic sample rate, returns `(audio, raw_consumed)`. Thread polls every `TICK_SEC` seconds.
7. **`stop_recording`** — transcribes the **entire** recording. Frontend replaces live segments with clean final transcript on `transcribe-done`.
8. **CLI mode preserved** — `main.rs` checks `--file` arg. No `--gui` → launches Tauri. Both paths compiled in single binary.

### Frontend (TypeScript/React)
1. **Pure view layer** — all logic in Rust commands. Frontend only invokes commands and listens to events.
2. **View routing** — `App.tsx` context (`AppCtx`) provides `view` ("landing" | "file" | "record"), `models`, `status`, `outputText`, `modelChoice`. Three views rendered conditionally.
3. **Common Layout** — `Layout` component wraps both `FileTranscribe` and `LiveRecord` with shadcn `SidebarProvider` + `Sidebar` + `SidebarInset`. Props: `sidebar`, `children`, `errorMessage`.
4. **System theme** — `prefers-color-scheme: dark` media query in CSS. No manual toggle, no `.dark` class.
5. **File exclusions from Biome** — `src-tauri/**`, `src/components/ui/**`, `src/hooks/**` are excluded.
6. **Auto-generated files** — shadcn components in `src/components/ui/` and `src/hooks/` are regenerated by `pnpm dlx shadcn@latest add <component>`. Do not edit them manually.

## Common Patterns

### Adding a new Tauri command
1. Define the function in `commands.rs` with `#[tauri::command]`
2. Define event payload structs with `#[derive(Serialize, Clone)]` if needed
3. Register in `lib.rs` `invoke_handler(tauri::generate_handler![...])`
4. Add TypeScript wrapper in `lib/commands.ts` using `invoke()`
5. Add event listener wrapper in `lib/events.ts` using `listen()` if streaming

### Emitting progress from a background thread
```rust
std::thread::spawn(move || {
    // ... do work ...
    let _ = app_handle.emit("my-event", MyPayload { ... });
});
```

### Reading AppState in a background thread
```rust
let state = app_handle.state::<AppState>();
let guard = state.recorder.lock();
```

## Commands & Events Reference

### Tauri Commands (12)
| Command | Args | Returns | Notes |
|---|---|---|---|
| `check_models` | — | `MissingModels` | Sync |
| `download_models` | — | `()` | Async, emits `download-progress` + `download-done` |
| `get_models_dir` | — | `String` | Sync |
| `pick_audio_file` | — | `Option<String>` | Sync, `rfd` dialog |
| `pick_save_file` | — | `Option<String>` | Sync, `.txt` filter |
| `pick_audio_save_file` | — | `Option<String>` | Sync, `.wav` filter |
| `transcribe_file` | `path, model` | `()` | Async thread, emits `transcribe-*` events |
| `start_recording` | `model, save_audio_path?, save_transcript_path?, vad_enabled?` | `()` | Spawns recording thread |
| `stop_recording` | — | `String` | Stops + transcribes, emits `transcribe-done` |
| `cancel` | — | `()` | Sets cancel flag |
| `save_text_file` | `path, content` | `()` | Writes text to disk |

### Events (7)
| Event | Payload | Emitted by |
|---|---|---|
| `download-progress` | `{ label, downloaded, total }` | `download_models` |
| `download-done` | `()` | `download_models` |
| `transcribe-progress` | `{ progress, text }` | File transcription |
| `transcribe-done` | `string` | File + recording transcription |
| `transcribe-error` | `string` | File + recording transcription |
| `record-progress` | `{ elapsed, text }` | Live recording segment |
| `record-segment` | `{ text }` | Live recording segment |
